# Project Manager

一个现代化的跨平台项目管理工具，聚合了项目运行、Git 管理、Node 版本管理、项目文件整理与系统集成功能。  
桌面端基于 `Tauri v2 + Vue 3 + TypeScript` 构建，同时提供 `uTools` / `ZTools` 插件版本。

---

## 核心能力

### 1. 项目管理与命令运行

- 统一管理多个前端或通用项目
- 支持单个或批量添加项目，也支持拖拽目录导入
- 自动识别 `package.json` 脚本，并支持自定义命令
- 一键在编辑器、终端、资源管理器中打开项目
- 命令运行日志支持持续输出，并在高频日志场景下自动跟随到底部

![命令运行](docs/images/command-runner.png)

### 2. Git 管理

- 内置 Git 图形界面，支持状态、差异、提交、历史、分支、远程仓库等操作
- 支持文件级与 hunk 级暂存 / 取消暂存 / 丢弃
- 支持并排 / 行内差异视图
- 支持提交历史查看、文件变更查看与区块回滚
- Git 工作区多处分栏支持拖拽调整，并会记住布局，下次打开自动恢复

![Git 管理](docs/images/git-panel.png)

### 3. 项目备忘录与文件管理

- 每个项目都可以维护 Markdown 备忘录
- 支持编辑 / 预览 / 分屏模式
- 可关联常用文件或文件夹，支持快速预览图片与文本

![项目备忘录](docs/images/project-memo.png)

![项目文件管理](docs/images/project-files.png)

### 4. Node 版本管理

- 可视化管理 NVM 中的 Node.js 版本
- 支持安装、卸载、切换版本
- 支持设置系统默认 Node
- 支持识别系统已有 Node，或手动添加自定义 Node 可执行路径

![Node 版本管理](docs/images/node-manager.png)

![安装 Node 版本](docs/images/node-install.png)

### 5. 端口管理

桌面端新增端口管理能力，适合排查本地开发时的端口占用问题。

- 查看当前正在占用的 TCP / UDP 端口
- 查看端口对应的 PID、进程名、程序路径与命令行
- 支持按进程结束占用端口的程序

说明：

- 当前端口管理为桌面端功能
- `uTools` / `ZTools` 插件环境下不提供端口管理

### 6. AI 提交信息生成

- 基于暂存区差异自动生成提交信息
- 支持 OpenAI 兼容接口
- 可配置 Base URL、API Key、模型和提示词模板
- 默认遵循 Conventional Commits 风格

![AI 提交信息生成](docs/images/ai-commit-settings.png)

### 7. 系统集成

- 支持多编辑器配置，项目可单独指定编辑器
- 自动检测系统终端，也支持自定义终端
- 支持托盘、关闭行为、自启动、右键菜单等系统能力
- 提供设置导入导出，便于迁移与备份

![设置页面](docs/images/settings.png)

## 快速开始

```bash
# 安装依赖
npm install

# 启动桌面端开发环境
npm run tauri dev

# 构建桌面端
npm run tauri build

# 构建 uTools 插件
npm run build:utools

# 构建 ZTools 插件
npm run build:ztools
```

### 1.0.9

**新功能**

- **终端打开后自动显示 Node 版本**：打开终端后自动执行 `node -v`，确认当前激活的 Node.js 版本，支持所有终端类型（PowerShell、pwsh、CMD、Windows Terminal、Cmder、Git Bash、gnome-terminal、konsole、xfce4-terminal、alacritty、kitty 等）
- **自动安装缺失 Node 版本**：项目配置了特定 Node 版本但尚未安装时，打开终端或运行脚本会自动触发安装并提示进度
- **nvm-windows 旧版兼容**：自动处理旧版 Node（如 v12）仅包含 `node64.exe` 的情况，通过硬链接创建 `node.exe` 确保 PATH 可正常解析

**优化**

- **Node 版本解析优先使用 NVM 固定路径**：项目终端和脚本运行现在优先匹配 NVM 安装目录中的版本专属路径，避免依赖会随全局切换变化的系统符号链接
- **插件版本同步更新**：uTools 和 ZTools 插件的终端功能同步支持 `node -v` 自动输出和 `node64.exe` 兼容

---

### 1.0.8

**新功能**

- **Node 项目自动补齐安装依赖命令**：新增、导入和历史已保存的 Node 项目都会自动补上一条“安装依赖”命令，并根据 `npm`、`yarn`、`pnpm`、`cnpm` 自动选择对应安装命令

**优化**

- **内置命令支持多语种显示**：自动生成的“安装依赖”命令改为使用内置标识按当前语言显示，切换中英文后，项目卡片、命令运行标签和运行日志都会同步显示对应语言
- **英文命令名称改为短标签**：英文环境下“安装依赖”显示为更紧凑的 `Install Deps`，减少项目卡片和标签页的占用宽度

**修复**

- **命令运行页大量输出滚动空白**：移除易出错的日志高度估算虚拟滚动，修复终端输出过多时向上翻动出现大片空白的问题
- **查看历史输出时被自动拉回底部**：新增“贴底跟随”判断和“回到底部”按钮，手动翻看旧日志时不再被新输出强制打断

---

### 1.0.7

**优化**

- **Git 长操作改为后台执行**：推送、拉取、抓取、状态刷新、历史读取等耗时操作改为后台执行，并在前端先渲染加载态，避免远端响应慢时窗口看起来无响应
- **网络请求补齐超时保护**：AI 请求、更新检查、Node 版本列表等外部接口统一增加超时与中断处理，接口长时间不返回时会及时给出错误提示，不再无限等待
- **AI 请求支持流式输出开关**：AI 设置新增流式输出配置，默认启用流式接收，兼容部分网关在非流式模式下返回空内容的情况

**新功能**

- **Git 长操作支持取消**：推送、拉取、抓取以及按仓库地址克隆项目时支持主动取消，并会真正终止后端 Git 子进程，减少卡住后只能强退应用的情况

**修复**

- **推送代码耗时过长导致界面假死**：修复远端网络较慢或仓库较大时，Git 推送长时间阻塞后让应用看起来无响应的问题
- **多类长耗时接口持续阻塞**：同步修复项目扫描、终端探测、更新下载等链路在慢环境下可能拖住界面的情况

### 1.0.6

**新功能**

- **添加项目支持远程仓库初始化**：当选择的目录还不是 Git 仓库时，可在添加项目弹窗中直接配置远程仓库地址，先拉取远程分支列表，再选择目标分支拉取代码到当前空目录
- **拉取后自动识别项目类型**：仓库克隆完成后会再次解析 `package.json`、锁文件和 `.nvmrc`，自动识别为 Node 项目或通用项目，无 `package.json` 时默认归类为“其他项目”
- **跨端统一支持仓库拉取流程**：桌面版 Tauri、uTools 插件和 ZTools 插件都补齐了远程分支查询与按分支克隆能力，添加项目体验保持一致

**优化**

- **Node 版本下拉改为按打开时刷新**：添加项目和编辑项目弹窗不再只在首次挂载时读取 Node 版本列表，安装、卸载或切换默认 Node 后，重新打开弹窗即可看到最新可选版本
- **项目元数据增强**：项目配置新增仓库地址、所选分支和是否通过仓库流程初始化等信息，便于后续导入导出和配置迁移时保留上下文

**修复**

- **拖拽 / 启动参数导入项目类型判断错误**：修复了导入没有 `package.json` 的目录时仍被错误识别为 Node 项目的问题
- **Git 页面错误提示自动消失**：Git 管理页中的操作报错改为需要手动关闭，避免在阅读较长错误信息时被自动收起
